**Til:** [API-leverandør, f.eks. api@gs1.org, developer@sallinggroup.com]

**Emne:** Søknad om API-tilgang til produktdata - Forbrukerrettet matvare-skanneapp

---

Hei,

Vi utvikler en forbrukerrettet mobilapp for mat- og drikkevarer i Danmark som lar brukere skanne strekkoder (GTIN/EAN) for å få tilgang til produktinformasjon, ingredienslister, allergeninformasjon og næringsinnhold.

**Om appen:**
Appen hjelper danske forbrukere med å ta informerte valg om matvarer ved å gi rask tilgang til korrekt produktinformasjon. Dette er særlig viktig for personer med allergier eller spesielle kostholdsbehov.

**Behov for API-tilgang:**
Vi søker read-only tilgang til produktdata via deres API for å:
- Tilby korrekt, oppdatert produktinformasjon fra autoritative kilder
- Maksimere treffrate ved å kombinere flere datakilder
- Sikre høy datakvalitet gjennom validering fra flere kilder
- Prioritere autoritative kilder over community-drevne alternativer

**Bruk av API:**
- ✅ Read-only tilgang (kun GET-forespørsler)
- ✅ Produktinformasjon: navn, merke, ingredienser, allergener, næringsinnhold, bilder
- ✅ Validering av GTIN/EAN-koder
- ❌ Ingen scraping, prishenting, skriving eller markedsføring

**Fokus på trygghet:**
Forbrukere er avhengige av korrekt produktinformasjon, spesielt allergener og næringsinnhold. Ved å bruke autoritative kilder sikrer vi at informasjonen er oppdatert, korrekt og pålitelig.

**Teknisk:**
- Vi respekterer rate limits og API-vilkår
- Feature flags for gradvis aktivering
- Robust feilhåndtering og caching
- Open-source tilgjengelig ved forespørsel

Vi setter stor pris på muligheten til å samarbeide med dere. Hvis dere har spørsmål eller trenger mer informasjon, ta gjerne kontakt.
